    - prettyReport([warnFinding]) with ci=false: stream receives '⚠️' badge and yellow summary

  <action>
    Replace src/reporters/pretty.ts entirely. Import picocolors as `import pc from 'picocolors'`. Import `type { Finding } from '../types.js'`.

    Export `interface PrettyOptions { stream?: { write(s: string): void }; ci?: boolean; }`.

    Export `function prettyReport(findings: Finding[], opts?: PrettyOptions): void`.

    Implementation:
    - `const out = opts?.stream ?? process.stdout`
    - `const ci = opts?.ci ?? false`
    - If findings.length === 0 and !ci: `out.write(pc.green('No findings.\n'))`; return
    - Filter visible = ci ? findings.filter(f => f.severity === 'error') : findings
    - Group visible by file into a `Map<string, Finding[]>` (insertion order)
    - For each [file, group]: `out.write(pc.bold(file) + '\n')`, then for each finding: badge = `pc.red('❌')` if error else `pc.yellow('⚠️ ')`; emit `  ${badge} ${f.file}:${f.line}  [${f.ruleId}]  ${f.message}\n` + `      ${pc.dim(f.remediation)}\n`
    - Always emit summary (even in ci mode): count errors/warns from ALL findings (not just visible); `${total} finding${total !== 1 ? 's' : ''} (${errors} error${errors !== 1 ? 's' : ''}, ${warns} warning${warns !== 1 ? 's' : ''})` — colored pc.red if errors > 0, pc.yellow if warns-only, pc.green if 0
    - picocolors auto-detects NO_COLOR and CI env vars; no manual process.env check needed

    Write tests/reporters/pretty.test.ts using `import { vi, describe, it, expect, beforeEach } from 'vitest'` and a mock stream: `const chunks: string[] = []; const stream = { write: (s: string) => { chunks.push(s); } };`. Reset chunks in beforeEach. Assert with `chunks.join('')` containing expected substrings using `toContain`.
