    Create src/signatures/rh002.ts. Same imports as rh001.ts.

    Export `function rh002(files: ParsedFile[], ctx: RepoContext): Finding[]`.

    Two helper functions (no export):
    - `function isStrongAssertion(content: string): boolean` — tests these patterns in order: `/\.toBe\([^)]+\)/`, `/\.toStrictEqual\(/`, `/\.toThrow\(['"\`]/`, `/\.toEqual\([^)]+\)/`, `/assertEqual\(.*,.*\)/`, `/assertIn\(/`, `/assertRaises\(/`, `/assert\s+.+\s*==\s*.+/`
    - `function isWeakAssertion(content: string): boolean` — tests: `/\.toBeDefined\(\)/`, `/\.toBeTruthy\(\)/`, `/\.toBeNull\(\)/`, `/\.toEqual\(\)/`, `/\.toThrow\(\)(?!\s*['"`(])/`, `/assertTrue\(/`, `/assert True/`, `/^\+\s*pass\s*$/`

    Main loop: for each file, for each chunk, collect `dels = chunk.changes.filter(c => c.type === 'del')` and `adds = chunk.changes.filter(c => c.type === 'add')`. For each del where `isStrongAssertion(del.content)`, find first add where `isWeakAssertion(add.content)`. If found, push Finding with: ruleId 'RH002', severity 'error', file: `file.to ?? file.from ?? ''`, line: `(add as { ln: number }).ln`, message: `Assertion weakened from ${extractPattern(del.content)} to ${extractPattern(add.content)}.`, remediation: 'Restore the specific value assertion to preserve test coverage strength.'

    Helper `extractPattern(content: string): string` — return first regex match group from content or a short snippet: trim leading `+/-` and surrounding whitespace, return first 30 chars.

    Write tests/signatures/rh002.test.ts. Use the same fixtureDiff helper and ctx stub as rh001.test.ts. Fixture test: run on RH002 fixture, normalise, assert matches expected.json (line 6). Add inline-diff test: construct a ParsedFile with one chunk containing a del line `"-  expect(add(1, 2)).toBe(3);"` and an add line `"+  expect(add(1, 2)).toBeDefined();"` with add.ln=6; assert rh002 returns one finding with line 6.
