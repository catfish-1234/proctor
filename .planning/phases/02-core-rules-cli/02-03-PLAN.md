    Check these patterns against `change.content` in order, pushing a Finding if any match:

    6. `/^\+\s*@skip\b/` — bare @skip decorator
    7. `/^\+\s*#.*def test_/` — commented-out Python test function

    For each match, build the message using a helper `buildSkipMessage(content: string): string` that returns a descriptive one-liner. For `.skip`, extract test name if possible: `/\.(skip|only)\s*\(\s*['"`](.*?)['"`]/` → `"Test '${name}' was disabled with .${modifier}."`. For other patterns a generic message is fine.

    Do NOT restrict to isTestFile — Python skip decorators can appear in any file that has tests. This matches the REQUIREMENTS.md description and the broader pattern set.

    Create src/signatures/rh007.ts. Same imports as rh001.ts.

    Export `function rh007(files: ParsedFile[], ctx: RepoContext): Finding[]`.

    Private helper `function isConfigFile(path: string): boolean` — return true if path matches:
    - `/(?:jest|vitest)\.config\.[mc]?[jt]s$/` — jest.config.ts, vitest.config.mjs, etc.
    - `/tsconfig(?:\.[^/]*)?\.json$/` — tsconfig.json, tsconfig.base.json, etc.
    - `/(?:pytest\.ini|setup\.cfg|pyproject\.toml|conftest\.py)$/`

    Private helper `function isConfigExclusionPattern(content: string): boolean` — return true if content matches any of:
    - `/testPathIgnorePatterns/`
    - `/testMatch\b/` (scope-narrowing)
    - `/testRegex\b/`
    - `/"exclude"\s*:/` (tsconfig)
    - `/norecursedirs/`
    - `/ignore\s*=/`
    - `/testpaths\s*=/`
    - `/collect_ignore/`

    Main loop: iterate ALL files (not only test files — config files are rarely in test globs). If `!isConfigFile(file.to ?? file.from ?? '')` skip. Otherwise, for each chunk, for each change where `change.type === 'add'`, if `isConfigExclusionPattern(change.content)`: push Finding({ ruleId: 'RH007', severity: 'error', file: file.to ?? file.from ?? '', line: (change as { ln: number }).ln, message: buildConfigMessage(change.content, file.to ?? file.from ?? ''), remediation: 'Remove testPathIgnorePatterns entry added in this change.' }).

    `buildConfigMessage(content: string, file: string): string` — extract the pattern name from content (trim `+`, first non-whitespace word or key that matched) and return `"Test path ignore pattern added to ${path.basename(file)} excluding ${extractedValue}."` where extractedValue is the first quoted string in content or 'test files'.

    Note: RH007 is unconditional in Phase 2 — no check for co-located code changes. Add a comment: `// ponytail: unconditional detection per Phase 2 design; cross-file correlation deferred to Phase 4`.

    Write tests/signatures/rh007.test.ts with:
    - Fixture test: run on RH007 fixture via spawnSync, normalise, assert matches expected.json (line 2)

    - Del-line test: config file ParsedFile with del change containing 'testPathIgnorePatterns'; assert []
    - tsconfig test: file.to='tsconfig.json', add line with '"exclude":'; assert Finding
