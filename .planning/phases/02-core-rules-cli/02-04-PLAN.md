  <action>
    Create src/engine.ts. Imports: `import type { ParsedFile } from './diff.js'`, `import type { RepoContext, Finding, Severity } from './types.js'`, `import { signatures } from './signatures/index.js'`, `import micromatch from 'micromatch'`.

    Export `function runChecks(files: ParsedFile[], ctx: RepoContext): Finding[]`:
    1. `const raw = signatures.flatMap(sig => sig(files, ctx)).filter(f => ctx.enabled.includes(f.ruleId))`
    2. `const afterSuppression = applySuppression(raw, files)`
    3. `const afterIgnore = applyIgnorePatterns(afterSuppression, ctx.ignorePatterns ?? [])`
    4. `return applySeverityOverrides(afterIgnore, ctx.severity ?? {})`

    Private (NOT exported) `function applySuppression(findings: Finding[], files: ParsedFile[]): Finding[]`:
    - For each finding, locate its ParsedFile using endsWith path matching: `const norm = (p: string) => p.replace(/\\/g, '/')`, then find the first file where `norm(file.to ?? file.from ?? '').endsWith(norm(finding.file)) || norm(finding.file).endsWith(norm(file.to ?? file.from ?? ''))`
    - If no file found: keep the finding (can't suppress what we can't locate)
    - Scan all changes across all chunks of the located file for a change whose effective line number equals `finding.line - 1`. Effective line: type === 'del' → `(c as {ln:number}).ln`; type === 'add' → `(c as {ln:number}).ln`; type === 'normal' → `(c as {ln1:number}).ln2`
    - If found, strip the leading diff char from change.content (`content.replace(/^[ +\-]/, '')`) and match against `/proctor-ignore:\s*(\S+)\s+reason:\s*(.+)/`. If match[1] === finding.ruleId AND match[2]?.trim() is non-empty: return false (suppress)
    - Default: return true (keep)

    Private `function applyIgnorePatterns(findings: Finding[], patterns: string[]): Finding[]`:
    - If patterns.length === 0: return findings
    - Return `findings.filter(f => !micromatch.isMatch(f.file.replace(/\\/g, '/'), patterns))`

    Private `function applySeverityOverrides(findings: Finding[], overrides: Record<string, Severity>): Finding[]`:
    - If no keys: return findings
    - Return `findings.map(f => overrides[f.ruleId] ? { ...f, severity: overrides[f.ruleId]! } : f)`

    Note: export `applySuppression` ONLY if needed for direct testing, otherwise test via `runChecks` with real mock data. Prefer testing the full `runChecks` pipeline with crafted inputs that exercise each post-processing step.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| diff context lines → suppression comment parser | Attacker-controlled diff content could craft a suppression comment |
| proctor.config.json ignorePatterns → micromatch | User-supplied glob patterns run through micromatch |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-07 | Tampering | applySuppression — crafted comment bypasses finding | accept | Suppression is a developer-intentional override; the format requires reason: text, making accidental suppression unlikely. Malicious suppression by the agent is the threat proctor is designed to detect in the agent's behavior, not in the diff itself. |
| T-02-08 | DoS | micromatch on large ignorePatterns array | accept | Patterns come from trusted proctor.config.json; large pattern sets are a developer misconfiguration, not an attack |
| T-02-SC | Tampering | npm/pip/cargo installs | accept | No new packages; micromatch already installed and audited in Phase 1 |
</threat_model>
