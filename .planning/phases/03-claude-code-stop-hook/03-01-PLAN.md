Implement the Claude Code Stop hook and its installer.

Output: `src/cli.ts` gains `stop-hook` subcommand and a fully-implemented `install-claude-hook` action. `tests/cli.test.ts` gains smoke tests covering both commands.

| CLI user → install-claude-hook --global | User can write to their own homedir; no privilege escalation surface |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | stop-hook stdin cwd field | mitigate | cwd passed as spawnSync cwd option, never shell-interpolated; JSON.parse in try/catch; invalid JSON falls back to process.cwd() |
| T-03-02 | Tampering | install-claude-hook settings.json write | accept | writes only to deterministic paths: process.cwd()/.claude/settings.json or homedir()/.claude/settings.json; no user-controlled path component |
| T-03-03 | Denial of Service | stop-hook stdin hang | mitigate | process.stdin.isTTY guard prevents hang on manual invocation without pipe; async iterator closes on EOF |
| T-03-04 | Information Disclosure | settings.json clobber on parse failure | mitigate | catch block leaves settings = {} only on ENOENT or corrupt JSON; warn is implicit (empty settings merge still adds proctor entry; pre-existing fields from corrupt JSON are lost but this is an edge case the user controls) |
| T-03-SC | Tampering | npm/pip/cargo installs | accept | no new package installs in this phase; slopcheck N/A |
